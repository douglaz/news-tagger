# Feature: Dependencies, Key Parsing, and Constructor Refactor

## Description
Add the `k256` and `bech32` crate dependencies, implement the `parse_secret_key` function for hex and `nsec1` Bech32 key formats, refactor `NostrPublisher` to store a `SigningKey` instead of a raw `SecretString`, make `new()` return `Result<Self>` with fail-fast validation, and update the CLI call-site to match the new return type. This feature lays the structural foundation for real cryptographic signing without yet changing event serialization or signature generation.

## Acceptance Criteria
- [ ] `k256 = { version = "0.13", features = ["schnorr"] }` and `bech32 = "0.11"` are added to `[workspace.dependencies]` in the root `Cargo.toml`.
- [ ] `k256` and `bech32` are wired as workspace deps in `crates/adapters/Cargo.toml`.
- [ ] A `parse_secret_key(raw: &str) -> Result<k256::schnorr::SigningKey>` function exists that: trims whitespace; decodes `nsec1`-prefixed input via Bech32 requiring HRP `nsec` and exactly 32-byte payload; otherwise parses 64-char lowercase/uppercase hex to 32 bytes; constructs a `SigningKey` and rejects zero or out-of-range scalars; returns clear error messages for every failure mode.
- [ ] `NostrPublisher` stores `signing_key: Option<k256::schnorr::SigningKey>` instead of `secret_key: SecretString`. The `secrecy` dependency is no longer used by this module (though may remain in `Cargo.toml` for other modules).
- [ ] `NostrPublisher::new(secret_key: SecretString, relays: Vec<String>) -> Result<Self>` parses the secret via `parse_secret_key` and returns `Err` immediately for invalid keys (empty, malformed hex, wrong hex length, wrong `nsec` HRP, wrong `nsec` payload length, invalid scalar).
- [ ] `NostrPublisher::disabled()` remains infallible, sets `signing_key: None`.
- [ ] CLI call-site at `crates/cli/src/commands/run.rs` line 232 is updated from `Ok(NostrPublisher::new(...))` to `NostrPublisher::new(...)` (or equivalent) to match the new `Result` return type.
- [ ] Both valid hex and valid `nsec` Bech32 encodings of the same 32-byte secret produce identical `SigningKey` values (and therefore identical derived public keys).
- [ ] Unit tests cover: hex parsing, `nsec` parsing, hex/nsec equivalence, and an invalid-key rejection matrix (empty string, 63-char hex, 65-char hex, non-hex chars, `nsec1` with wrong HRP like `npub1`, `nsec1` with 31-byte payload, `nsec1` with 33-byte payload, all-zero scalar).
- [ ] The project compiles without errors (`cargo check --workspace`).
- [ ] Existing tests that use `NostrPublisher::new` or `NostrPublisher::disabled` are updated to compile and pass.
